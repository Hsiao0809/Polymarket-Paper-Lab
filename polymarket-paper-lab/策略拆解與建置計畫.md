## 一、先修正對那篇貼文的解讀

貼文作者的路徑（1 月接觸 → 3 月 $300 起 → 5 月 $17,000）容易被讀成「學會 vibe coding 就能賺錢」。實際上他文中已經寫清楚了失敗與成功的分界：

| 階段 | 市場 | 結果 | 原因（他自己說的）|
|---|---|---|---|
| 前期 | BTC 15 分鐘漲跌 | **虧損**，十幾種策略全部「跑幾天正收益就下墜」 | 莊家多、機器人執行龜速 |
| 後期 | 美股單日漲跌 | 獲利 | 不需低延遲、無人操控、可用期權鏈定價 |

**結論：他不是找到了神策略，他是換到一個「自己的弱點不重要」的戰場。** 這是整篇最可遷移的一課。

他的收益來源有兩條，兩條都不是「預測能力」：
1. **跨市場定價價差** — 用高效率的期權市場資料，去定價低效率的預測市場合約
2. **Maker 獎勵** — 掛單返佣，結構性收入，與方向無關

### 必須誠實面對的部分

- $300 → $17,000 是 **56 倍**，四個月。這是極端離群值。我們讀到這篇文章正是因為他成功了；用同樣方法失敗的人不會發文。**倖存者偏差**。
- 成本他也寫了：四個月 **$1,200** 在訂閱與 API 上，加上大量時間。這是實質的固定成本。
- 他的紀錄是今年 1–5 月。**市場結構此後已經改變**（見下節）。

## 二、實測：市場結構已經改變

### 2.1 目標市場仍然存在

透過 Gamma API 查詢確認，今天（2026-07-21）仍有：

- `SPY (SPY) Up or Down on July 21?`（結算 2026-07-21 20:00 UTC）
- `SPY Daily Up or Down`（系列）
- 週/月級別的 S&P 500 區間市場

### 2.2 手續費：從無到有

Polymarket 原本免手續費。變更時間軸：

- **2026 年 1 月** — Crypto 類別開始收 taker fee
- **2026 年 3 月 30 日** — 擴大到幾乎所有類別，含 **Finance**
- Geopolitical / World events 類別目前仍免費

從 API 直接讀到的 stocks 類別參數：

```json
"feesEnabled": true,
"feeType": "general_fees",
"feeSchedule": { "exponent": 1, "rate": 0.04, "takerOnly": true, "rebateRate": 0.25 }
```

**三個關鍵性質：**

1. `takerOnly: true` — **Maker 完全不付手續費**
2. `rate: 0.04` — taker 費率，費用在 p=0.5 附近最高，往兩端遞減
3. `rebateRate: 0.25` — taker 付的費用有 **25% 每日回饋給 maker**

貼文作者三月起跑，正好踩在這個轉折點上。**他的經驗有一部分來自免手續費時代，不能直接照搬。**

## 三、資料指向的結論（與直覺相反）

一般人讀那篇貼文會想去複製「吃別人的錯誤定價」。但現在的費用結構說明：

- **主動吃單（taker）現在要付 4% 費率** → 你的定價優勢必須先大到覆蓋這筆費用，才開始賺錢
- **掛單（maker）零手續費 + 分享 $1,000/天獎勵池 + 拿 25% taker 費回饋**

> **結構性收益在掛單那一側，不在吃單那一側。**

### 定價模型的角色因此改變

不再是「找到錯誤定價 → 衝進去吃掉」，而是：

> **決定我的限價單該掛在哪個價位，才不會被逆選擇。**

**逆選擇（adverse selection）是做市唯一的真風險**：你的掛單總是在你判斷錯誤的那一刻成交。SPY 要大漲時，買你的 NO；要大跌時，買你的 YES。防禦這件事，正是 Black-Scholes + 期權鏈隱含波動率的用途——它給你一個獨立於 Polymarket 盤口的公允值參考。

### 資本門檻的觀察

獎勵資格最小 200 股 × 現價 $0.82 ≈ **$164**（單邊）。雙邊掛約需 **$300+**。

**貼文作者起手的 $300 大概不是任意選的數字，那接近能吃到 maker 獎勵的最低門檻。**

目前帳戶餘額 $2.35，距離這個門檻還有很大距離。

## 四、建置計畫（先驗證，後投錢）

### 階段 0：資料管線（無風險，先做）

- [ ] 接 Gamma API，每分鐘拉取 SPY 單日盤的盤口、成交、獎勵參數
- [ ] 接一個免費或低成本的期權資料源，取得 SPY 同日到期的期權鏈
- [ ] 落地成時序資料庫（SQLite 就夠）

**驗收標準**：能連續 5 個交易日無中斷記錄完整盤口與期權資料。

### 階段 1：定價模型與驗偽（關鍵階段）

- [ ] 用期權鏈的隱含波動率，推算「SPY 今日收盤高於開盤」的機率
  - 注意：需處理**日盤/夜盤波動率差異**（貼文作者踩過的坑）
  - 注意：單日到期的期權有極端 gamma 效應，IV 取值要謹慎
- [ ] 逐分鐘比對「模型機率」vs「Polymarket 中價」
- [ ] **量測 gap 的分布：平均多大？多常出現？扣掉 4% taker 費後還剩多少？**

> **這一步的目的是把策略殺掉，不是證明它可行。**
> 如果 gap 中位數小於手續費，吃單這條腿直接砍掉，只留做市那條。

**驗收標準**：產出一張 gap 分布圖與統計摘要，能明確回答「edge 是否存在、多大」。

### 階段 2：做市模擬（紙上交易）

- [ ] 以模型公允值為中心，模擬雙邊掛單
- [ ] 模擬成交（保守假設：只有當市價穿過你的價位才成交）
- [ ] **必須模擬逆選擇**：計算成交後到結算的實際盈虧，而非假設賺到價差
- [ ] 套用實際獎勵公式估算獎勵收入（需查 Polymarket 獎勵計分規則）

**驗收標準**：至少 20 個交易日的模擬權益曲線，含「價差收益 / 獎勵收益 / 逆選擇損失」三項拆解。

### 階段 3：決策點

只有在階段 2 顯示 **扣除逆選擇後仍為正** 時，才討論是否投入真實資金，以及金額。

## 五、已知風險清單

| 風險 | 說明 |
|---|---|
| **逆選擇** | 做市的核心風險。成交總發生在你錯的時候 |
| **獎勵池稀釋** | $1,000/天是與所有合格 maker 按比例分。參與者變多，你的份額變少 |
| **獎勵政策可變** | 這是平台補貼，隨時可能調降或取消。不可視為永久收入 |
| **Edge 衰減** | 3¢ 價差存在正因為還沒人做。同類文章傳播後會變窄 |
| **單一市場集中** | 全押 SPY 單日盤，市場一旦下架或改規則即歸零 |
| **固定成本** | 期權 API、訂閱費。低資本規模下可能吃掉全部收益 |
| **監管/地域** | polymarket.com 在部分地區無法直接存取（本次瀏覽器實測即失敗）|

## 六、下一步

建議從**階段 0** 開始，因為它零成本、零風險，而且產出的資料無論策略成不成立都有價值。

階段 1 是真正的決策點：**它會告訴你該不該繼續，而不是告訴你能賺多少。**
